## Garbage Criteria Used For This Analysis

A file was considered a true garbage candidate only if it met one or more of these conditions with evidence:

- clearly useless technical residue with no retrieval value;
- empty or effectively empty content;
- metadata shell with no structural role;
- duplicate wrapper that adds no indexing, navigation, or maintenance value;
- obsolete leftover whose presence harms the corpus more than it helps it;
- misplaced file whose content is already safely represented elsewhere in the active corpus and whose current copy has no staging or audit value.

A file was **not** treated as garbage merely because it was:

- small;
- overview-style;
- a staging candidate;
- duplicated in historical git state;
- awkwardly placed inside `dirty/`.
